Demo tras aplikacji Javalin

Poniżej znajdują się endpointy zdefiniowane w pliku HelloJavalin.java wraz z krótkim opisem działania.

Metoda Adres Opis Demo
GET /hello Zwraca prostą odpowiedź tekstową Hello World!. Otwórz
GET /headers Pokazuje nagłówek User-Agent; dodatkowo wypisuje wszystkie nagłówki w konsoli serwera. Otwórz
GET /user Zwraca przykładowy obiekt użytkownika w formacie JSON. Otwórz
GET /boom Celowo rzuca wyjątek, aby pokazać obsługę błędów (HTTP 500). Otwórz
POST /echo Odczytuje body żądania i odsyła je jako Echo: .... Zobacz test POST
GET /users/{id}/posts/{postId} Przykład parametrów ścieżki; zwraca id użytkownika i postId (z walidacją typu liczbowego). Otwórz przykład
GET /search?q=...&limit=... Przykład parametrów zapytania; limit domyślnie ma wartość 42. Otwórz przykład
GET /admin/users Trasa konkretna dla sekcji administracyjnej (użytkownicy). Otwórz
GET /admin/* Trasa ogólna (wildcard) dla pozostałych adresów zaczynających się od /admin/. Otwórz przykład

Jak przetestować POST /echo

Najprościej: użyj formularza poniżej i wyślij dane bezpośrednio z przeglądarki.

Dla terminala możesz użyć także curl:

curl -X POST http://localhost:8080/echo -d "To jest test POST"
